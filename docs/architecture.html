memory_rs System Architecture

OpenZ-inspired Rust-native MCP Memory Server featuring 5 Cognitive Layers & SQLite storage

1. Central Orchestrator

  • • Coordinator maps all core CRUD tasks securely to respective tables.
  • • Enables multi-layer search query dispatching.
  • • Integrates model-driven embeddings with local execution logic.
  • • Performs weighted scoring rank calculation dynamically.

2. SQLite Database Schema

  • • graph_nodes: Maps entity key to serialized JSON observations.
  • • graph_edges: Maps directed connections between entities.
  • • semantic_metadata: Stores raw fact texts, importance scores, and embedding blobs.
  • • episodic_logs: Logs execution stages, steps taken, errors, and reflections.
  • • codebase_signatures: Indexes file items, signatures, and imports.

3. Composite Ranking Model

  • • Calculates cosine similarity against vector databases.
  • • Applies exponential temporal decay over elapsed hours (λ = 0.01).
  • • Integrates manual importance coefficients (1.0 - 5.0).
  • • Dynamically incorporates historical success rate for optimal reflections.
